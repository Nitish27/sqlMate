<task type="auto">
  <name>Task 2: Fix test_connection to use options builder pattern and add cargo audit CI</name>
  <files>src-tauri/src/core/connection_manager.rs, .github/workflows/security.yml</files>
  <action>
1. In `connection_manager.rs`, find the `test_connection` method (around lines 64-110). It currently uses URL string interpolation:
   ```rust
   let url = format!("postgres://{}:{}@{}:{}/{}", user, pass, host, port, db);
   ```
   Replace with the options builder pattern for each database type:

   **PostgreSQL:**
   ```rust
   use sqlx::postgres::{PgConnectOptions, PgSslMode};
   let opts = PgConnectOptions::new()
       .host(host)
       .port(port)
       .username(user)
       .password(pass)
       .database(db);
   let mut conn = sqlx::postgres::PgConnection::connect_with(&opts).await?;
   ```

   **MySQL:**
   ```rust
   use sqlx::mysql::MySqlConnectOptions;
   let opts = MySqlConnectOptions::new()
       .host(host)
       .port(port)
       .username(user)
       .password(pass)
       .database(db);
   let mut conn = sqlx::mysql::MySqlConnection::connect_with(&opts).await?;
   ```

   **SQLite:**
   SQLite uses file paths, not user/pass. Keep the existing pattern for SQLite (it doesn't have the password-in-URL problem). Use `SqliteConnectOptions::new().filename(path)` if not already.

   Ensure the test_connection still pings after connecting to verify the connection is alive.

2. Also check the `connect` method (lines 46-62). It likely already uses PgPoolOptions/MySqlPoolOptions (confirmed in RESEARCH). If it still has URL interpolation anywhere, fix it too.

3. Create `.github/workflows/security.yml`:
   ```yaml
   name: Security Audit

   on:
     push:
       branches: [main]
     pull_request:
       branches: [main]
     schedule:
       - cron: '0 6 * * 1'  # Weekly Monday 6am UTC

       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - name: Install cargo-audit
           run: cargo install cargo-audit --locked
         - name: Run cargo audit
           working-directory: src-tauri
           run: cargo audit
   ```

4. Run `cargo build` to confirm the test_connection changes compile.

5. Run `cd src-tauri && cargo audit` locally if cargo-audit is installed (if not, skip — CI will catch it).
